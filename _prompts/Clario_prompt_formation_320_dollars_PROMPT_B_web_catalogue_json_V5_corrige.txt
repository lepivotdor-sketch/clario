======================================================================

La formation doit respecter cette structure simple :
- catalogue : formations.html ;
- page modèle unique : formation.html ;
- fichier de données : data/formations.json ;
- chargement par ID unique ;
- URL : formation.html?id=[slug].

- ne crée pas de fichier HTML individuel pour chaque formation ;
- le champ fichier_html doit toujours être « formation.html » ;
- le champ id doit être identique au slug ;
- le slug doit être unique, propre, lisible, en ASCII minuscule + tirets uniquement, sans accent, sans souligné, ≤ 60 caractères, et commencer par le préfixe de catégorie ;
- l’URL recommandée doit toujours être formation.html?id=[slug] ;
- le JSON doit être directement compatible avec le catalogue ;
- ne mets jamais de virgule finale dans le JSON ;
- aucun emoji dans le JSON ;
- les notes doivent être des nombres entiers entre 0 et 100 ;
- prix_numerique doit être un nombre entier (jamais une chaîne) ;
- prix_affiche et prix_numerique doivent être cohérents (ex. « 25 $ » ↔ 25) ;
- les tags doivent être un tableau de 5 à 10 chaînes minuscules sans doublon, sans accent, sans #.

Règles SEO obligatoires :
- titre ≤ 60 caractères ;
- sous_titre ≤ 90 caractères ;
- resume_court ≤ 220 caractères (servira aussi de méta-description, donc ≤ 155 caractères si possible) ;
- promesse ≤ 140 caractères.

Règles typographiques :
- espace insécable (caractère Unicode U+00A0) entre le nombre et le symbole $ dans tout le contenu et dans les chaînes du JSON (ex. « 25\xa0$ ») ;
- guillemets droits pour le JSON, guillemets typographiques (« » ou « ») dans le contenu client ;
- aucun emoji dans le JSON ; emojis tolérés dans le contenu client uniquement, max 2 par section.
